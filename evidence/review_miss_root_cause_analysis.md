### Anti-Regression Checklist
- Runtime-evidence gaps to block on
  - Do not allow `SUCCESS`/`PASS` if a blocker-class row still says `NEEDS_TEST`, `BLOCKED`, or equivalent for concurrency, lock cleanup, retry-safety, or awaited-I/O behavior.
  - Require at least one live contention/recovery witness test for every behavioral lock invariant.
  - Require explicit downstream owner assignment before any retained runtime-proof debt crosses a gate.
- Dead-export classification rules
  - Any symbol documented as public API must be classified as external contract, compatibility shim, or internal-only helper.
  - A gate that audits named product surfaces only must also assert that no extra documented public helpers remain unclassified.
  - "Not in current MCP surface matrix" is not enough to close a symbol-level ambiguity; the review must say whether the symbol is intentionally non-public.
- Dependency/manifest-clarity checks
  - For every user-visible dependency claim, bind package authority, import authority, and manifest/header wording to one canonical tuple.
  - Reject split-proof reviews where packaging tests, runtime imports, and surface docs each pass independently but describe different authorities.
  - If manifest text defers discoverability to another surface (for example `tela_list_providers`), the gate must verify that the referenced surface is also the authoritative runtime source for that fact.
